## 一、整体设计理念

### 1.1 核心思想

数据驱动方法的核心是将**参数辨识与实时控制解耦**：

| 传统EKF方法 | 数据驱动方法 |
|------------|-------------|
| 每个周期微调参数 | 周期性批量更新参数 |
| 参数与状态耦合估计 | 状态估计用固定参数 |
| 单步优化 | 全局优化 |
| 在线连续学习 | 批量离线学习 |

### 1.2 设计原则

| 原则 | 说明 |
|-----|------|
| **解耦原则** | 状态估计与参数辨识分离，降低复杂度 |
| **质量优先** | 只用高质量数据辨识，保证精度 |
| **渐进更新** | 参数变化幅度受限，防止剧烈跳变 |
| **验证保障** | 新参数必须通过验证才能使用 |
| **版本可溯** | 保留参数历史，支持回滚 |

## 二、数据采集模块设计

### 2.1 数据来源与内容

#### 输入数据

| 数据项 | 来源 | 采样频率 | 说明 |
|-------|------|---------|------|
| $P_{in}(t)$ | 控制器输出 | 高频(10-20Hz) | 加热功率 |
| $T_s(t)$ | 热敏电阻 | 高频(10-20Hz) | 传感器温度 |
| $T_{env}(t)$ | 环境温度传感器 | 低频(1Hz) | 环境温度 |
| $T_c(t)$ | 散热器温度传感器 | 低频(1Hz) | 冷端温度 |

#### 衍生数据

| 数据项 | 计算方式 | 用途 |
|-------|---------|------|
| $\dot{T}_s$ | 数值微分或滤波估计 | 动态特征识别 |
| $\dot{P}_{in}$ | 差分计算 | 激励强度判断 |
| $T_h^{est}, T_b^{est}$ | 状态观测器输出 | 扩展训练目标 |

### 2.2 数据质量评估体系

#### 质量分级标准

**第一级：有效性检查**

| 检查项 | 标准 | 不通过后果 |
|-------|------|-----------|
| 温度范围 | $50°C \leq T_s \leq 400°C$ | 丢弃 |
| 功率范围 | $0 \leq P_{in} \leq P_{max}$ | 丢弃 |
| 变化率合理性 | $|\dot{T}_s| < 10°C/s$ | 丢弃 |
| 时间连续性 | 无重复、无缺失 | 丢弃 |

**第二级：质量分级**

| 等级 | 判定标准 | 信息价值 |
|-----|---------|---------|
| **A级（高质量）** | $|\dot{T}_s| \geq 0.5°C/s$ 或 $|\dot{P}_{in}| \geq 5W/s$ | 高，动态激励强 |
| **B级（中等）** | $|\dot{T}_s| \geq 0.1°C/s$ 或 $|\dot{P}_{in}| \geq 1W/s$ | 中，有动态信息 |
| **C级（低质量）** | 准稳态，温度基本不变 | 低，仅稳态信息 |
| **D级（无效）** | 不满足有效性检查 | 丢弃 |

### 2.3 数据价值评估

不同数据点对参数辨识的贡献不同，需要评估其"价值"：

#### 价值因子

| 因子 | 计算方式 | 物理意义 |
|-----|---------|---------|
| **动态价值** | $w_{dyn} = f(|\dot{T}_s|, |\dot{P}_{in}|)$ | 动态激励越强，信息量越大 |
| **温度价值** | $w_T = f(T_s)$ | 边界温度区域数据更稀少，价值更高 |
| **工况价值** | $w_{mode} = f(\text{加热/冷却})$ | 加热过程信息量通常大于冷却 |
| **误差价值** | $w_{err} = f(\text{预测误差})$ | 误差大的区域需要更多样本 |

### 3.2 滑动窗口机制

#### 为什么需要滑动窗口？

| 原因 | 说明 |
|-----|------|
| **时效性** | 系统特性可能随时间变化，旧数据可能失效 |
| **内存限制** | 无法无限存储历史数据 |
| **适应性** | 窗口滑动可追踪参数漂移 |

#### 窗口设计

```
时间轴：
├──── t-window ────┼──── t-window/2 ────┼──── t ────┤
│                  │                    │           │
│   旧数据区       │    过渡区          │  新数据区  │
│   (权重衰减)     │    (正常权重)      │  (高权重)  │
│                  │                    │           │
└──────────────────┴────────────────────┴───────────┘

窗口属性：
├── 时间跨度：可配置（如最近10-30分钟数据）
├── 最小数据量：触发更新前必须达到的量
└── 淘汰策略：先进先出(FIFO)或按价值淘汰
```

#### 时间加权策略

数据权重随时间衰减：

$$w_{time}(t) = \exp\left(-\frac{t_{now} - t}{\tau}\right)$$

其中 $\tau$ 为时间常数，控制衰减速度。

### 3.3 温度分布均衡

#### 问题：数据分布不均

正常运行中，系统大部分时间工作在目标温度附近的稳态，导致：
- 中温区（如200°C）数据过剩
- 低温区和高温区数据稀少
- 动态过程数据稀少

将温度范围划分为若干区间，每个区间设置最小数据量要求：

## 四、参数辨识算法设计

### 4.1 辨识问题数学建模

#### 完整模型回顾

$$\frac{dT_h}{dt} = \theta_8 \cdot P_{in} - \theta_1(T_h - T_b)$$

$$\frac{dT_b}{dt} = \frac{\theta_1}{\theta_2}(T_h - T_b) - \theta_3(T_b - T_s) - \theta_5(T_b - T_{env}) - \theta_6(T_b - T_c) - \theta_7(T_b^4 - T_{env}^4)$$

$$\frac{dT_s}{dt} = \theta_4(T_b - T_s)$$

#### 优化问题定义

$$\min_{\theta} J(\theta) = \sum_{k=1}^{N} w_k \cdot \left[ T_s^{meas}(t_k) - T_s^{sim}(t_k; \theta) \right]^2$$

约束条件：
$$\theta_i^{lower} \leq \theta_i \leq \theta_i^{upper}, \quad i = 1, 2, ..., 8$$

其中：
- $w_k$ 为第 $k$ 个数据点的权重
- $T_s^{sim}$ 为模型仿真输出
- $N$ 为数据点总数

### 4.2 仿真模型求解

#### 数值积分方法

采用**四阶龙格-库塔法(RK4)**求解微分方程：

$$\begin{aligned}
k_1 &= f(x_k, u_k, \theta) \\
k_2 &= f(x_k + \frac{h}{2}k_1, u_k, \theta) \\
k_3 &= f(x_k + \frac{h}{2}k_2, u_k, \theta) \\
k_4 &= f(x_k + hk_3, u_k, \theta) \\
x_{k+1} &= x_k + \frac{h}{6}(k_1 + 2k_2 + 2k_3 + k_4)
\end{aligned}$$

#### 初始状态确定

仿真起始点需要确定初始状态 $x_0 = [T_h^0, T_b^0, T_s^0]^T$：

| 策略 | 说明 | 适用场景 |
|-----|------|---------|
| **使用测量值** | $T_s^0 = T_s^{meas}$，$T_h^0 = T_b^0 = T_s^0$ | 数据量少时 |
| **使用状态估计** | 使用EKF估计的 $T_h^{est}, T_b^{est}$ | 数据量充足 |
| **分段优化** | 每段独立优化初始状态 | 精度要求高 |

### 4.3 优化算法选择

#### 算法对比

| 算法 | 优点 | 缺点 | 适用场景 |
|-----|------|------|---------|
| **Levenberg-Marquardt** | 收敛快，局部搜索能力强 | 易陷局部最优 | 初值较好时 |
| **差分进化(DE)** | 全局搜索，不依赖初值 | 计算量大 | 初值不确定时 |
| **粒子群(PSO)** | 全局搜索，易并行化 | 参数敏感 | 多核平台 |
| **CMA-ES** | 自适应强，鲁棒性好 | 计算量大 | 高精度需求 |
| **混合策略** | 结合全局+局部 | 实现复杂 | 推荐方案 |

#### 推荐混合策略

```
第一阶段：全局探索（差分进化）
├── 目的：找到参数空间中的优良区域
├── 种群数：20-50
├── 迭代次数：50-100
└── 输出：近似最优解 θ_rough

第二阶段：局部精调（Levenberg-Marquardt）
├── 目的：在优良区域内精确收敛
├── 初始点：θ_rough
├── 迭代次数：50-100
└── 输出：精确最优解 θ_optimal
```

### 4.4 多目标优化扩展

考虑单一目标可能导致参数偏向某一工况，可引入多目标：

$$\min_{\theta} \mathbf{J}(\theta) = \begin{bmatrix} J_{dynamic}(\theta) \\ J_{steady}(\theta) \\ J_{regularization}(\theta) \end{bmatrix}$$

其中：
- $J_{dynamic}$：动态段预测误差
- $J_{steady}$：稳态段预测误差
- $J_{regularization} = \|\theta - \theta_{prior}\|^2$：正则化项（防止偏离物理先验）

### 5.2 物理约束验证

#### 参数范围约束

| 参数 | 物理意义 | 下界 | 上界 | 约束来源 |
|-----|---------|------|------|---------|
| $\theta_1$ | $K_{hb}/C_h$ | 0.1 | 2.0 | 时间常数0.5-10秒 |
| $\theta_2$ | $K_{hb}/C_b$ | 0.01 | 0.5 | 时间常数2-100秒 |
| $\theta_3$ | $K_{bs}/C_b$ | 0.01 | 0.2 | 时间常数5-100秒 |
| $\theta_4$ | $K_{bs}/C_s$ | 0.05 | 1.0 | 时间常数1-20秒 |
| $\theta_5$ | $K_{conv}/C_b$ | 1e-4 | 0.1 | 对流换热范围 |
| $\theta_6$ | $K_{hc}/C_b$ | 1e-4 | 0.1 | 传导换热范围 |
| $\theta_7$ | $K_{rad}/C_b$ | 1e-12 | 1e-8 | 辐射换热范围 |
| $\theta_8$ | $1/C_h$ | 0.05 | 1.0 | 热容范围1-20 J/K |

#### 参数关系约束

| 约束 | 表达式 | 物理意义 |
|-----|--------|---------|
| 热容比值 | $1 < \theta_1/\theta_2 < 20$ | 加热块热容应大于加热芯 |
| 散热比例 | $\theta_5 + \theta_6 < 0.1$ | 总散热率上限 |
| 传热比例 | $\theta_1 > \theta_2$ | $C_b > C_h$ |

#### 变化幅度约束

防止参数剧烈跳变：

$$\left| \frac{\theta_i^{new} - \theta_i^{old}}{\theta_i^{old}} \right| < \delta_i$$

| 参数类型 | 最大变化幅度 | 原因 |
|---------|-------------|------|
| $\theta_5, \theta_7$（散热） | 30% | 环境可能变化较快 |
| $\theta_1 \sim \theta_4$（传热） | 15% | 接触热阻变化较慢 |
| $\theta_6$（喉管传导） | 10% | 相对稳定 |
| $\theta_8$（热容倒数） | 5% | 物理不变，仅补偿误差 |

### 5.3 交叉验证设计

#### 数据划分策略

```
完整数据集
    │
    ├── 随机划分 ──────────────────────────────┐
    │                                          │
    ▼                                          ▼
训练集 (80%)                               验证集 (20%)
│                                          │
│ 用于参数优化                             │ 用于精度验证
│                                          │
└──────────────────────────────────────────┘
```

#### 验证指标

| 指标 | 公式 | 说明 |
|-----|------|------|
| **RMSE** | $\sqrt{\frac{1}{N}\sum(T_s^{meas}-T_s^{sim})^2}$ | 均方根误差 |
| **MAE** | $\frac{1}{N}\sum|T_s^{meas}-T_s^{sim}|$ | 平均绝对误差 |
| **最大误差** | $\max|T_s^{meas}-T_s^{sim}|$ | 极端误差 |
| **R²** | $1 - \frac{\sum(T^{meas}-T^{sim})^2}{\sum(T^{meas}-\bar{T})^2}$ | 拟合优度 |

#### 验证通过标准

$$\text{RMSE}_{new} < \text{RMSE}_{old} \times (1 - \epsilon)$$

其中 $\epsilon$ 为最小改进阈值（如5%），确保新参数确实有改进。

或使用更宽松的标准：

$$\text{RMSE}_{new} < \text{RMSE}_{old} \times (1 + \delta)$$

其中 $\delta$ 为容许退化（如2%），允许小幅波动。

### 5.4 实时验证（可选）

对于安全要求高的场景，可采用短期试运行验证：

```

### 7.3 参数切换策略

#### 无缝切换设计

参数更新不应影响正在进行的控制：

```
时间线：
├──── t_1 ────┼──── t_2 ────┼──── t_3 ────┼──── t_4 ────┤
│             │             │             │             │
│  使用 θ_old │  参数优化中  │  验证中     │  使用 θ_new │
│  (正常运行)  │  (θ_old)    │  (θ_old)    │  (正常运行) │
│             │             │             │             │
│             │ 后台计算    │ 后台计算    │ 原子替换    │
│             │ 不影响控制   │ 不影响控制   │             │
│             │             │             │             │
└─────────────┴─────────────┴─────────────┴─────────────┘

关键：
├── 参数优化在后台线程运行
├── 主控制回路始终使用 θ_current
├── 验证通过后原子替换
└── 替换瞬间完成，不影响控制
```

#### 参数平滑过渡（可选）

如果担心参数突变影响控制，可采用平滑过渡：

$$\theta(t) = \theta_{old} + (\theta_{new} - \theta_{old}) \cdot \alpha(t)$$

其中 $\alpha(t)$ 为过渡函数：

| 过渡方式 | $\alpha(t)$ | 特点 |
|---------|-------------|------|
| 阶跃 | $\alpha = 1$ (瞬时) | 简单，可能有跳变 |
| 线性 | $\alpha = t/\tau$ | 平滑，简单 |
| 指数 | $\alpha = 1-e^{-t/\tau}$ | 自然，推荐 |

### 7.4 状态观测器设计

参数更新后，状态观测器应继续正常工作：

#### 方案：简化EKF（仅估计状态）

```
状态方程（参数固定）：
  dT_h/dt = θ_8·P_in - θ_1·(T_h - T_b)
  dT_b/dt = θ_1/θ_2·(T_h - T_b) - θ_3·(T_b - T_s) - ...
  dT_s/dt = θ_4·(T_b - T_s)

测量方程：
  y = T_s

估计状态：T_h, T_b（T_s直接测量）
```

#### 参数更新对观测器的影响

| 情况 | 影响 | 处理方式 |
|-----|------|---------|
| 参数小幅变化 | 观测器继续收敛 | 无需处理 |
| 参数较大变化 | 可能暂时发散 | 重置协方差矩阵P |
| 参数突变 | 估计偏差 | 可重新初始化状态 |

### 7.5 MPC预测模型更新

MPC的预测模型使用当前参数 $\theta_{current}$：

#### 预测方程

$$\mathbf{x}_{k+1} = A(\theta_{current}) \mathbf{x}_k + B(\theta_{current}) u_k$$

#### 参数更新后的处理

```
参数更新时刻 t_update：
│
├── 1. 替换参数：θ_current ← θ_new
│
├── 2. 更新MPC预测矩阵：
│      A_new ← compute_A(θ_new)
│      B_new ← compute_B(θ_new)
│
├── 3. 重新计算预测时域：
│      X_pred ← predict(A_new, B_new, x_current, U_sequence)
│
└── 4. 继续MPC优化循环
```

### 8.2 回滚触发条件

| 条件 | 说明 | 回滚版本 |
|-----|------|---------|
| RMSE突然增大 | 预测误差超过阈值 | 上一有效版本 |

```

│
├── 检测异常
│   ├── 预测误差：RMSE > 3×历史均值

│
├── 判断严重程度
│   ├── 轻度：继续监控
│   ├── 中度：准备回滚
│   └── 重度：立即回滚
│
└── 执行回滚
    ├── 选择目标版本（通常是上一有效版本）
    ├── 原子替换
    ├── 记录回滚原因
    └── 暂停后续更新一段时间
```

├── 参数异常
│   ├── 超出范围 → 约束优化或拒绝
│   ├── 变化过大 → 降低学习率或拒绝
│   └── 频繁波动 → 增加稳定性约束
│
└── 更新异常
    ├── 连续失败 → 暂停更新，检查原因
    ├── 验证不通过 → 检查数据或优化算法
    └── 计算超时 → 简化问题或增加资源
```
